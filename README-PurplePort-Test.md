# PurplePortGreeter TEST

## Що робить ця версія

- працює окремо від оригінального userscript;
- за замовчуванням використовує `PREPARE_ONLY`;
- відкриває форму та заповнює тему і текст;
- **не натискає Send автоматично**;
- обробляє лише 1 профіль за один тестовий запуск;
- не записує підготовлене повідомлення як надіслане;
- після перезавантаження відновлює прогрес тільки в режимі Pause;
- використовує безпечніший пошук посилання профілю;
- перевіряє наявність теми, iframe і кнопки Send;
- додає події `input/change/blur`;
- виводить результат у Chrome DevTools Console.

## 1. Завантаження у GitHub

1. Створіть приватний репозиторій, наприклад `purpleport-greeter-test`.
2. Натисніть **Add file → Upload files**.
3. Завантажте `PPgreeter.TEST.user.js`.
4. Натисніть **Commit changes**.
5. Відкрийте файл у GitHub і натисніть **Raw**.

Для приватного репозиторію Tampermonkey зазвичай не може стабільно оновлювати файл через Raw без авторизації. Для першої перевірки найпростіше скопіювати код із GitHub у Tampermonkey вручну.

## 2. Встановлення у Tampermonkey

1. Відкрийте Tampermonkey → Dashboard.
2. Вимкніть оригінальний `PureplePortGreeter`, щоб два скрипти не працювали одночасно.
3. Натисніть **Create a new script**.
4. Видаліть стандартний код.
5. Вставте весь вміст `PPgreeter.TEST.user.js`.
6. Натисніть `Ctrl+S` / `Cmd+S`.
7. Переконайтеся, що `PurplePortGreeter TEST` увімкнений.

## 3. Безпечний перший тест

У файлі має залишатися:

```js
mode: AUTOMATION_MODE.PREPARE_ONLY,
maxPreparedProfilesPerRun: 1,
enableAutomaticSend: false,
```

1. Авторизуйтеся у PurplePort.
2. Відкрийте пошук фотографів.
3. Натисніть `F12` → **Console**.
4. Натисніть Run у панелі скрипта.
5. Скрипт має відкрити форму одного профілю та заповнити її.
6. Перевірте отримувача, тему й текст.
7. У Console знайдіть `Profile result` та таблицю з `recipient`, `subject`, `bodyPreview`.
8. Кнопку Send натискайте лише вручну після перевірки.

## 4. Очікуваний результат

У Console має бути статус:

```text
prepared
```

і причина:

```text
Form filled; press Send manually after inspection.
```

Статус `sent` у тестовому режимі не повинен створюватися.

## 5. Якщо форма не заповнилася

Збережіть із Console:

- текст помилки;
- `subjectFound`;
- `frameFound`;
- `sendButtonFound`;
- скриншот відкритої форми;
- URL сторінки без приватних даних.

Не вмикайте `AUTO_SEND`, доки не визначено надійний сигнал успішної відправки PurplePort.
